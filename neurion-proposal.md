## 선택된 방향

### 방향: 합체 - Claude Code TUI Dashboard

**선택 이유**: Claude Code 자체가 터미널 환경이므로, 같은 터미널 내에서 화면 분할하는 TUI(cokacdir-main 스타일)가 가장 자연스러운 통합 방식. 4개 그룹을 하나로 합쳐 완전체 구성.

### 예상 핵심 기능 (초안, 3개)

| # | 기능 | 설명 | 관련 아이디어 |
|---|------|------|-------------|
| 1 | 화면 분할 TUI 엔진 | cokacdir-main 스타일로 터미널을 좌(대시보드)/우(Claude Code)로 분할. Textual/Rich 기반 | #1, #7 |
| 2 | TASKS.md 파싱 + Hook 이벤트 수집 | TASKS.md 실시간 파싱 및 Claude Code Hook에서 이벤트 스트림 수신. 에이전트 상태/진행률 데이터 수집 | #3, #4, #8 |
| 3 | 시각화 위젯 (에이전트 맵, 프로그레스, 에러) | 에이전트 트리 뷰, 전체 프로그레스 링, 에러/경고 하이라이트를 대시보드에 위젯으로 배치 | #6, #12, #13 |

### 예상 타겟 사용자

- **Claude Code 헤비 유저**: 오케스트레이션/에이전트 스킬을 만들어 대규모 프로젝트에 활용하는 개발자
- **AI 코딩 워크플로우 관리자**: 여러 에이전트의 실행 상태를 한눈에 파악하고 싶은 사람
